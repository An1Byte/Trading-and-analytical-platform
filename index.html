Volumes FORTS & MICEX-RTS:

Volumes FORTS & MICEX-RTS is a mode for representing instruments of the Russian market using cluster charts and volume charts. It is also possible to trade in one click and place limit or pending orders, if you first link the instrument in question with the instrument from your terminal.

Resources:

Resources - external and internal resources that can help in work and research. Everything is assembled into a single action game for maximum efficiency when surfing third-party portals.

Arbitrage offline:

Arbitrage offline is a mode of constructing synthetic instruments, in which it is possible to view both historical information on the "spread" and current ones as well. Synthetics are plotted based on the closing price levels of the bars of the instruments, depending on the "spread", in other words, the chart is presented as a curved line. There is also the ability to trade synthetics in one click.

Multicharts:

Multicharts is a view mode in which many selected charts are displayed in one area, while the initial common point for plotting the charts is assigned to the instruments, and at this point all instruments are brought to a total imaginary value, due to which the level of correlation at any moment in time relative to a given point will be 100% correct and will not shift (change) when scrolling. However, you can also set your own contango / backwardation ratio for each instrument. There is also a one-click opportunity to conclude transactions at a time for each set of instruments participating in the presentation.

Arbitrage online:

Arbitrage online is a mode of building synthetic instruments, in which the history is accumulated as long as the portal and terminals are synchronized with each other. The data on the "spread" between the two instruments is as accurate as possible, due to the candlestick representation of the synthetic chart. This mode is more suitable for arbitrageurs and traders engaged in pair trading on instruments between which the correlation level is in the range from 80% to 99%. There is also the ability to trade synthetics in one click.

Arbitrage versions:

Arbitrage versions is a mode of constructing synthetic instruments, in which it is possible to view both historical information on the "spread" and current ones as well. Synthetics are plotted based on the closing price levels of the bars of the instruments, depending on the "spread", in other words, the chart is presented as a curved line. A distinctive feature of this mode, versus Arbitrage offline mode, is that the "spread" is built depending on how the volatility of the dependent instruments changes, so the final result is a glued cut of synthetics with different levels of volatility. There is also the ability to trade synthetics in one click.

Portal and terminals not synchronized

This section allows the user to build a multi-chart from the N-th set of instruments, while, due to the determination of the reference point, as well as equating all the selected instruments to the price of the first one, it avoids such bugs as, for example, displacement of charts relative to each other when scrolling and scaling the chart window ... To build a multi-chart - select the tools you need for plotting, while the number of charts superimposed on each other is unlimited. Then specify the start date of drawing, after which, click 'BUILD' to build the multiplot. If you have not yet synchronized the portal with the terminals, then, of course, you should first start with this, and for this, click on the "SYNCHRONIZE WITH TERMINALS" button, after which the server will generate an ID for you. You just need to copy it and paste it into the 'ID' field in the expert settings of your terminal, and then start the expert. When connecting multiple terminals, the procedure is the same - the same identifier is assigned to each terminal. There is no point in remembering this identifier, since a new identifier will be allocated to you on the next request. When the portal is refreshed or exited - the ID is automatically no longer valid and will need to be resynchronized, so use only the portal buttons for any transitions - everything is there to efficiently iterate through all layers of the application. Please note that the current ID is broadcast in the status line, so you can use it in case the clipboard has been overwritten with other data ... In order for your terminal to interact with the portal, you must first download, install and run the appropriate expert. The expert is distributed free of charge and is available for download on this portal. To get an expert - click on the \ "DOWNLOADS \" button.

Thanks to the Online mode, you can build synthetics from any two instruments, while you can observe the exact intra-bar dynamics of changes for the created instrument, which can be especially useful when trading on small divergences. This is the advantage of this regime - the presence of dynamics of changes. The disadvantage of this mode is that it is not possible to view the history of such a chart, however, you can accumulate history on your own by simply waiting. Select from the list 2 instruments for which you want to build synthetics for arbitrage or pair trading, set the coefficients for them, or the system itself will select the most optimal of them and click the 'BUILD' button. Correlation coefficients for automatic calculation require entering the number of bars on the basis of which the coefficients will be calculated, which are determined as the ratio of the arithmetic average differences of the high and low prices of each bar in a given area multiplied by the ratio of the tick sizes of the 1st and 2nd instrument, and also multiplied by the ratio of the value of ticks of the 1st and the second instruments. The odds should be selected in such a way that the theoretically open oppositely directed positions on both instruments compensate each other as much as possible. If the coefficient for the first instrument, for example, is equal to 1.2, and for the second one, then, in order to sell the resulting instrument, you will need to sell 1.2 * n contracts (lot) of the first one and buy 1 * n contracts (lot) of the second, and if you want to buy the resulting instrument, then, on the contrary, buy 1.2 * n contracts (lot) of the first and sell 1 * n contracts (lot) of the second, where n is the volume of a position in contracts (lot) concluded for each deal. The resulting instrument is usually called a spread or a synthetic instrument. The spread between two instruments is the difference between the bid values of the first instrument and the bid of the second, but when making such a deal, you should take into account the size of the spread!
